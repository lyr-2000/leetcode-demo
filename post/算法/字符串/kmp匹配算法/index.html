KMP 匹配算法

预处理过程：

  1. pre_4 == last_4
  2. pre_3 == last_3
  3. pre_2 == last_2

如果不是最长的后缀，就可能错过答案的位置，要保证后缀和前缀最长。

证明 两个串可以直接匹配：

$$
TA == TB\

SA == TA\

so:\
SA == TA\
TA == SB\

$$

Ta 可以和 SB直接对齐。

直接 比较 sb后面黄色的那段就可以了，蓝色的那段不需要重复进行匹配。 黄色的那段位置是 i+j

我们令 母串 S 的黄色位置是 i, 模式T当前匹配位置是 j,
如果 i,j 不匹配，j就往前跳， i变，j下标变化，往模式串前面跳 $j=next[j]$。

$Tb==Sb ==Ta$

如果 Tb后面和Sb无法匹配，那么就从 Ta开始和 Sb匹配【模式串往前挪动,母串不变。】

如果相等，就相当于 i,j增长一位，匹配下一位。

如果不能匹配，Ta模式串里面还有更小的串可以进行匹配，我们叫做 小Ta。

如果我们可以预处理出 Ta 和 Tb的信息的话，我们就可以提升字符串匹配的效率。

KMP 预处理的过程

kmp预处理的是 j 跳到j' ,j' 跳到 j'' 的整个过程。 整个过程我们可以理解为是递归的。

首先getNext 是模式串和模式串比较， 如果 模式串 i位置不等于 j位置，j就往前跳。

如果匹配成功， 说明第 i位可以跳到 第 j+1 位置。

void GetNext(const char*pattern,int *next) {
    next[0] = -1;
    for (int i=1,j=-1;pattern[i];i++) {
        while(j!=-1 && pattern[j+1] != pattern[i]) j = next[j];
        if(pattern[j+1] == patern[i]) j++;
        // 无法往前跳，或者 已经匹配完成
        next[i] = j;
    }
    return;
}

KMP母串是不会往前回溯的，只会往后面跳，所以算法效率是非常高效的。

class Solution {
public:
    int getNextJ(vector<int> &next,string &a,string &b,int i,int j) {
        while(j!=-1 && a[i] != b[j+1])  j=next[j];
        if(a[i] == b[j+1]) j++;
        return j;
    }
    int strStr(string S, string T) {
        int n= S.size(),m = T.size();
        if(m==0) return 0;
        vector<int> next(m);
        next[0] = -1;
        int j=-1;
        for(int i=1;i<m;i++) {
            j = getNextJ(next,T,T,i,j);          
            next[i] =j;
        }
        j=-1;
        for(int i=0;i<n && j<m;i++) {
            j = getNextJ(next,S,T,i,j);
            // printf("%d,%d\n",i,j);
            if(j==m-1) return i-j;
        }
        
        return -1;
    }
};
